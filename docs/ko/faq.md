# FAQ

## Q. VRC Emote를 사용하지 않는 아바타는 Emote Installer를 사용할 수 없나요?

네. 현재 `VRC Emote Installer`는 **`VRCEmote` 파라미터(1~8)**를 기반으로 동작하는 구조를 전제로 합니다.

따라서 아래 중 하나의 방식으로 아바타를 준비해야 합니다.

- **직접 `VRCEmote` 파라미터를 사용하는 Emote 메뉴/Action 레이어 구조를 구성**하거나
- **VRChat 기본(Default) Action 레이어**를 사용하도록 아바타를 수정해야 합니다.

> 즉, 레디얼 메뉴에서 Emote를 선택했을 때 `VRCEmote` 값이 바뀌며 동작하는 구조가 필요합니다.

## Q. `Setup VRC Emote` 버튼을 눌렀는데 스테이트 설정이 제대로 적용되지 않는 것 같아요.

일부 아바타는 Action 레이어 구조가 복잡해 자동 추적이 실패할 수 있습니다. 예를 들어:

- `VRCEmote`를 사용하는 분기가 **여러 군데에 중복**되어 있는 경우
- **복잡한 Sub StateMachine** 구조로 인해 시작/종료 흐름을 안정적으로 찾기 어려운 경우

이 경우 다음 순서로 해결하는 것을 권장합니다.

1. **`Start Action State` / `End Action State` 이름을 직접 입력**해 주세요.  
   (자동 추적이 실패해도 수동 지정으로 병합할 수 있습니다.)

2. Emote 분기가 **Sub StateMachine 내부에 존재**한다면,  
   고급 옵션에서 **액션 루트 추적(merge scope)**을 활성화한 뒤  
   `Action Merge Scope(액션 서브머신 Root)` 경로/이름을 직접 지정해 주세요.

> 요약: “자동 설정이 안 된다” → 대부분은 **스테이트 이름 수동 입력 + merge scope 지정**으로 해결됩니다.

## Q. 제 창작물을 이 도구로 모듈화해서 상업적으로 배포해도 되나요?

가능합니다. 다만 **이 도구(패키지) 자체를 포함해 재배포하는 형태**는 권장하지 않으며,  
사용자가 정식 경로(VCC 등)로 설치할 수 있도록 안내하는 방식을 권장합니다.

---

## Q. 문제 발생이나 손해에 대한 지원은 어떻게 되나요?

본 도구 사용으로 인해 발생한 문제/손해에 대해서는, 고의 또는 중대한 과실이 있는 경우를 제외하고
책임을 지지 않으며 손해배상 의무도 부담하지 않습니다.

다만 개인 개발 프로젝트 특성상, 가능 범위 내에서 피드백 수용 및 유지보수를 위해 노력합니다.
